# Case Studies

Use this folder for sanitized examples that show how the public-memory patterns work in practice.

Good case studies should:

- demonstrate a reusable architecture or workflow pattern
- remove client-sensitive and infrastructure-sensitive details
- focus on lessons learned rather than raw chronology
- link back to architecture, workflows, prompts, or decisions that explain the general method

Avoid:

- raw transcripts
- operational secrets
- screenshots with private details
- examples that depend on hidden internal context to make sense
